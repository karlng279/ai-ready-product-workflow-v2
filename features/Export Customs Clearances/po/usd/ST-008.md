# ST-008: Acceptance Criteria

**Story:** Create support record from booking  
**USL Reference:** [../usl.md#st-008-create-support-record-from-booking](../usl.md#st-008-create-support-record-from-booking)  
**Last Updated:** 2026-05-13

---

## UI Elements

- **AC-001:** Operations users can identify ECC support records created from completed bookings.
- **AC-002:** The support record displays booking reference, customer, lane, vessel/voyage when available, and cutoff timing when available.

---

## UI Behavior

- **AC-003:** A support record is created after booking completion with a valid ECC request.
- **AC-004:** Operations users can open the support record without asking the customer to repeat core booking details.
- **AC-005:** If support record creation fails, the ECC request is flagged for operational review.

---

## Logic

- **AC-006:** Each valid ECC booking creates no more than one active support record.
- **AC-007:** Support record creation stores booking ID, customer ID, ECC request time, support case ID, and SLA due time.
- **AC-008:** SLA due time is calculated as 24 hours after booking completion unless an approved lane rule specifies otherwise.

---

## Special Notes

- **AC-009:** Support records must not expose customer data to users without appropriate operational access.

---

## Non-Functional Requirements

- **NFR-001:** Support record creation succeeds within 2 minutes for 95% of completed ECC bookings.
- **NFR-002:** Duplicate active support records occur for less than 0.5% of completed ECC bookings.
- **NFR-003:** Access to support records is restricted to authorized operational roles in 100% of tested permission cases.
