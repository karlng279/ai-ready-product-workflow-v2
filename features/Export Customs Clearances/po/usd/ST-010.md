# ST-010: Acceptance Criteria

**Story:** Manage blocked or unsupported support  
**USL Reference:** [../usl.md#st-010-manage-blocked-or-unsupported-support](../usl.md#st-010-manage-blocked-or-unsupported-support)  
**Last Updated:** 2026-05-13

---

## UI Elements

- **AC-001:** Operations users can mark an ECC support record as blocked.
- **AC-002:** Operations users can select a blocked reason from approved reason values.
- **AC-003:** Customers can receive or view a clear next-step message when support is blocked.

---

## UI Behavior

- **AC-004:** Marking a record as blocked requires a blocked reason.
- **AC-005:** When a case is blocked by missing customer information, the next-step message identifies the customer-owned action.
- **AC-006:** When a case is unsupported, the next-step message explains that ECC cannot proceed for the booking.

---

## Logic

- **AC-007:** Blocked cases are excluded from first support completion metrics unless a valid first support action was already recorded.
- **AC-008:** Unsupported cases must retain the original booking and ECC request references for audit.

---

## Special Notes

- **AC-009:** Unsupported messaging must not suggest that customs clearance is impossible; it only describes ECC service availability.

---

## Non-Functional Requirements

- **NFR-001:** Blocked or unsupported status updates within 5 seconds for 95% of saved changes.
- **NFR-002:** Approved blocked reasons are used in 100% of blocked support records.

---
