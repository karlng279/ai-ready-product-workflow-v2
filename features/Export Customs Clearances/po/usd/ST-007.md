# ST-007: Acceptance Criteria

**Story:** Confirm ECC request to customer  
**USL Reference:** [../usl.md#st-007-confirm-ecc-request-to-customer](../usl.md#st-007-confirm-ecc-request-to-customer)  
**Last Updated:** 2026-05-13

---

## UI Elements

- **AC-001:** The booking confirmation identifies that ECC was requested or selected.
- **AC-002:** The confirmation includes the expected first support action target.

---

## UI Behavior

- **AC-003:** After booking completion with ECC, the customer sees confirmation without needing to re-enter booking details.
- **AC-004:** The customer can access next-step information from the confirmation state.

---

## Logic

- **AC-005:** Confirmation only appears when the completed booking has a valid ECC request.
- **AC-006:** Confirmation includes booking reference and service request reference when available.

---

## Special Notes

- **AC-007:** Confirmation must state that customer-owned or broker-owned information may still be required.

---

## Non-Functional Requirements

- **NFR-001:** Confirmation displays within 1 second after booking completion for 95% of ECC bookings.
