# ST-002: Acceptance Criteria

**Story:** Communicate service availability  
**USL Reference:** [../usl.md#st-002-communicate-service-availability](../usl.md#st-002-communicate-service-availability)  
**Last Updated:** 2026-05-13

---

## UI Elements

- **AC-001:** Eligible customers see an ECC availability message during booking.
- **AC-002:** Ineligible customers do not see an ECC purchase or request option.

---

## UI Behavior

- **AC-003:** The availability message explains that ECC is optional.
- **AC-004:** The customer can continue booking when ECC is unavailable.

---

## Logic

- **AC-005:** The availability state reflects the latest eligibility decision for the current booking.
- **AC-006:** Availability messaging does not display if eligibility fails for unsupported lane or operations readiness.

---

## Special Notes

- **AC-007:** Availability language must not imply that ECC is required to complete an export booking.

---

## Non-Functional Requirements

- **NFR-001:** Availability state renders within 1 second after eligibility is evaluated for 95% of sessions.
