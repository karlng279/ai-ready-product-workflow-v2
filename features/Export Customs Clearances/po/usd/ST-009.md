# ST-009: Acceptance Criteria

**Story:** Record first support action  
**USL Reference:** [../usl.md#st-009-record-first-support-action](../usl.md#st-009-record-first-support-action)  
**Last Updated:** 2026-05-13

---

## UI Elements

- **AC-001:** Operations users can record a first support action for an ECC support record.
- **AC-002:** The support record displays first support action status and timestamp.

---

## UI Behavior

- **AC-003:** Recording the first support action updates the support record status.
- **AC-004:** The support record shows whether the first action occurred within 24 hours.
- **AC-005:** Operations users cannot record a first support action without selecting an action type.

---

## Logic

- **AC-006:** First support action timing is calculated from booking completion time to first action timestamp.
- **AC-007:** Only the earliest valid first support action counts toward the 24-hour SLA metric.

---

## Special Notes

- **AC-008:** If the support action is blocked by missing customer information, the case must be marked with a blocked reason instead of counted as completed support.

---

## Non-Functional Requirements

- **NFR-001:** First support action status updates within 5 seconds for 95% of saved actions.
- **NFR-002:** SLA calculation accuracy is 100% in test cases using known timestamps.
